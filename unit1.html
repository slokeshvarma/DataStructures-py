Unit 1: Algorithm Analysis

1. Big-Oh Notation

Big-O notation is used to describe the performance or complexity of an algorithm. It provides an upper limit on the runtime of an algorithm as the input size grows. For example, linear search has O(n) complexity as it traverses the input once. 


# Example: O(n) linear complexity
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

2. Comparative Analysis

Comparative analysis involves comparing different algorithms based on time and space complexity. For example, binary search is more efficient than linear search for sorted arrays with a time complexity of O(log n).

3. Recursion

Recursion is a method where a function calls itself to solve a smaller version of a problem. Recursive algorithms are often used in divide-and-conquer problems, such as quicksort or finding factorials. 


# Example: Factorial using recursion
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)

print(factorial(5))  # Output: 120

4. Analyzing Recursive Algorithms

Recursive algorithms are analyzed by identifying their recurrence relations and solving them using methods like the Master Theorem. The complexity depends on the depth and size of each recursive call.
